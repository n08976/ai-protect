Pick the app (manifest), the SDLC stage, and optionally narrow to a single adapter. Inline help is under each control.

The app to scan. Each manifest declares its tier, data sensitivity, and surfaces — that's what the policy table consults to decide which adapters run.
The v2.1 SDLC stage that defines what runs. Each stage maps to a different adapter list per the policy table.
  • intake — manifest validation + shadow-AI recon (subfinder/httpx/naabu/katana) + OWASP Noir endpoint discovery.
  • design — threat model check (Tier 1/2 required) + IaC scan (Checkov) + guardrails validation.
  • build — the SAST / secrets / SCA bulk (Semgrep, Bandit, gosec, TruffleHog, Gitleaks, pip-audit, Grype, Syft, ModelScan, Presidio, Agentic Radar, garak/PyRIT lite). Fastest feedback loop.
  • preprod — the full gate: DAST (ZAP, Burp, Nuclei, sqlmap, Ride), MCP scope (highest-leverage), AI red team (garak/PyRIT full, eval_suite), adversary emulation (Atomic, Caldera), guardrails smoke.
  • production — telemetry drift + anomaly detection against live AI telemetry.
Default is build. Use preprod for the full gate before a deployment decision.
Scope within the stage you picked above.
  • all adapters — runs every adapter the policy table assigns to this tier × stage. This is the typical full re-scan and what you want when you've changed code, manifest, or want a fresh baseline.
  • single adapter — runs only that one tool. Faster; use it when you've updated an adapter's config, tuned a noisy rule (e.g. bandit B113), installed a new scanner binary, or want to validate one finding without re-running the whole stage. Adapters that don't fire at the selected stage will graceful-degrade as unavailable.
